## 1. 경쟁 구도 개요

MyDNA Insurance Agent는 단일 카테고리의 경쟁자를 갖지 않는다. 세 개의 기존 시장이 각각 일부 문제를 해결하지만, 어느 것도 "유전자 데이터 주권 보장 + 정밀 보장 설계 + 글로벌 유동성"을 동시에 제공하지 못한다.

경쟁자는 세 그룹으로 분류된다:

- **Group A**: 전통 보험 GA/플랫폼 (데이터 주권 없음)
- **Group B**: Web3 보험 프로토콜 (체인 사일로, 유전자 처리 없음)
- **Group C**: DTC 유전자 분석 서비스 (보험 연결 없음, 프라이버시 취약)

---

## 2. 그룹별 경쟁자 분석

### Group A — 전통 보험 GA / 인슈어테크 플랫폼

| 서비스 | 특징 | 한계 |
|---|---|---|
| **보맵 (Bomapp)** | 국내 1위 보험 비교 플랫폼, 200만+ 사용자 | 유전자 데이터 활용 없음, 통계 집단 평균 추천, 원본 데이터 서버 저장 |
| **카카오페이 보험** | 대규모 분산 채널, 간편 가입 | 맞춤 설계 불가, 유전 정보 기반 상품 없음 |
| **뱅크샐러드 보험** | DTC 유전자 제공 + 보험 비교 동시 운영 | 유전자 결과와 보험 추천이 단절됨, 원본 데이터 자사 서버 보관 |
| **Lemonade (US)** | AI 기반 언더라이팅, UX 혁신 | 유전자 데이터 없음, 미국 단일 시장 |

**공통 한계**: 사용자의 유전자 원본을 보험사 또는 플랫폼 서버에 보관. 분석 알고리즘이 불투명하며 정보 비대칭을 플랫폼이 독점.

---

### Group B — Web3 보험 프로토콜

| 서비스 | 특징 | 한계 |
|---|---|---|
| **Nexus Mutual** | 이더리움 기반, 스마트 컨트랙트 해킹 보험 | 유전자 데이터 없음, 이더리움 단일 체인, 개인 건강보험 불가 |
| **Etherisc** | 분산형 보험 프로토콜, 항공·작물 보험 | 헬스케어 적용 없음, DeFi 사용자 한정 |
| **InsurAce** | 멀티체인 포트폴리오 보험 | 금융 자산 보호에 특화, 개인 건강·유전자 보험 영역 없음 |
| **Chainvest** | Web3 보험 인프라 | 체인 사일로, 프라이버시 레이어 없음 |

**공통 한계**: DeFi 프로토콜 리스크 헤징에 특화되어 있어 개인 건강보험·유전자 맞춤 상품 설계와는 거리가 멈. 멀티체인 지원 부재 또는 초기 단계.

---

### Group C — DTC 유전자 분석 서비스

| 서비스 | 특징 | 한계 |
|---|---|---|
| **23andMe (US)** | 글로벌 최대 DTC 유전자 서비스, 1400만+ DB | 2023년 데이터 유출 사고, 서버 중앙 보관, 보험 연결 없음 |
| **젠톡 (GenTok)** | 마크로젠 운영, 175종 분석 | 건강 리포트 제공에 그침, 보험 상품 연결 없음 |
| **진스타일 (GeneStyle)** | 테라젠헬스, 건강 관리 특화 | 보험 연결 없음, 데이터 자사 서버 보관 |
| **Color Genomics (US)** | 임상 수준 유전자 분석 | B2B 고용주 채널 중심, 소비자 직접 보험 연결 없음 |

**공통 한계**: 유전자 분석 결과를 사용자에게 리포트로 제공하는 데서 끝남. 보험 상품 설계로 이어지지 않으며, 원본 데이터를 중앙 서버에 보관하여 데이터 주권 보장 불가.

## 4. "지금, 우리만 가능한 이유"

NEAR Protocol의 세 가지 핵심 기술이 **2026년 상반기에 동시에 사용 가능**해진 시점이 이 서비스의 존재 조건이다.

- **IronClaw Runtime** (TEE 격리 실행): 유전자 데이터를 하드웨어 수준에서 분석하고 즉시 삭제. 서비스 운영자도 열람 불가.
- **Confidential Intents** (2026.02 출시): 보험 계약 내용이 퍼블릭 멤풀에 노출되지 않음. 개인의 리스크 수치가 보험사에 전달되지 않고 "검증된 조건 충족"만 증명.
- **Chain Signatures**: NEAR 계정 하나로 이더리움·솔라나 기반 글로벌 보험 상품에 서명. 기존 Web3 보험이 단일 체인에 묶인 구조적 한계를 극복.

이 세 기술 중 하나라도 없으면 본 서비스는 성립하지 않는다. 2025년 이전에는 불가능했고, 2026년 NEAR 스택이 아니면 다른 체인에서도 불가능하다.
