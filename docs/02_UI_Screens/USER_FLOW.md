## 1. 개요 (Overview)
본 문서는 "MyDNA Insurance Agent" 자체 웹 DApp(MVP)의 핵심 사용자 여정(User Journey)을 정의합니다. 기존의 거대 금융사 앱 대비 **'완벽한 통제력(Data Sovereignty)'**과 **'안전함'**을 느끼도록 설계된 5단계 프론트엔드 흐름입니다.

---

## 2. 사용자 여정 시나리오 (5-Step User Journey)

### Step 1: 온보딩 및 신뢰 구축 (Onboarding & Trust)
- **화면 구성**: 랜딩 페이지 및 Web3 지갑 연결(NEAR Wallet Login)
- **사용자 행동**: '프라이버시 서약(Privacy Manifesto)' 확인 후 지갑 연결.
- **UX 포인트 (핵심 가치 전달)**: 기존의 복잡한 회원가입 절차 없이 블록체인 지갑만으로 연결됨을 강조. "당신의 핵심 유전자 데이터는 보험사가 아닌, 오직 당신의 통제 하에 둡니다"라는 카피와 자물쇠 애니메이션 배치.

### Step 2: 프라이빗 데이터 업로드 (The Vault)
- **화면 구성**: 개인 금고(Private Cloud) 뷰포트 및 파일 드래그 앤 드롭 영역.
- **사용자 행동**: DTC 업체(ex. 젠톡, 진스타일)에서 다운받은 유전자 결과 파일(VCF, TXT, PDF 등)을 업로드.
- **UX 포인트 (시각적 격리)**: 파일이 공용 서버가 아닌, 내 계정에 할당된 '개인 금고' 아이콘 속으로 안전하게 암호화되어 들어가는 연출(UI)을 통해 심리적 안정감 제공.

### Step 3: 투명한 격리 분석 + ZKP 증명 (In-TEE Analysis & Proof)
- **화면 구성**: TEE 분석 시각화 대시보드 (로딩 화면 역할).
- **사용자 상태**: NEAR AI 에이전트(IronClaw Agentic Harness)가 금고에 일시적으로 진입하여 데이터 해석 후 Noir ZKP proof 생성.
- **UX 포인트 (휘발 및 투명성)**:
  - 5단계 Progress 표시: "TEE 진입 중" → "데이터 분석 중" → **"ZKP 증명 생성 중 (수치 암호화)"** → "위험 프로파일 생성" → "원본 데이터 소각 완료"
  - ZKP 단계에서 "보험사에 전달될 내용: 자격 충족 여부만 (유전자 수치 미포함)" 문구 표시.
  - 리스크 분석이 끝나자마자 **"원본 복호화 데이터 즉시 파기(Memory Purged)"** 시각적 피드백(먼지 애니메이션)으로, 분석용 데이터가 서버에 잔류하지 않음을 증명.

### Step 4: 정밀 진단 및 추천 대시보드 (Diagnostics & Recommendation)
- **화면 구성**: 유전자 취약점 리포트 및 AI 맞춤 특약 추천 카드 배열.
- **제공 정보**: "췌장암 고도 위험", "당뇨군 평균 대비 주의 요망" 등의 직관적 요약.
- **유도 행동(CTA)**: 고객의 취약점 방어에 맞춰 AI가 패키징한 '최적 보험 조합 스나이핑 카드(가칭)' 확인 후 동의(Approve) 클릭.
- **UX 포인트**: 기존 보험 가입 시 나오는 불필요한 담보목록은 제거하고, "오직 나만의 유전자 맹점에 특화된" 보험이라는 것을 붉은색 경고와 푸른색 커버리지 디자인으로 명확히 대조.

### Step 5: 무자각 기밀 체결 (Confidential Checkout)
- **화면 구성**: 원클릭 서명 모달 (NEAR Wallet Signature).
- **사용자 행동**: 추천된 보험료를 NEAR, USDC 등으로 결제 서명. 이 서명은 **Chain Signatures MPC**(`v1.signer`)가 처리하며, 사용자는 NEAR 계정 하나로 서명.
- **UX 포인트**:
  - 결제 모달에 "ZKP 증명 첨부됨" 배지 표시 — 유전자 수치 대신 proof bytes가 보험사로 전달됨을 명시.
  - "보험사로 전송 중: 상세 지표 필터링 완료 (Confidential Intents)" 상태 노출.
  - 서명 진행 상태: "Chain Signatures로 서명 중" → "Confidential Intents 제출 중" → "온체인 확정 완료"
  - 완료 후: 트랜잭션 해시 + "보험사에 전달된 정보: 자격 충족 여부만 (유전자 수치 미포함)" 요약 표시.

## 3. 에러 케이스 및 엣지 케이스 흐름 (Error & Edge Cases)

### E-1: 지갑 연결 실패 (Step 1)
- **발생 조건**: 사용자가 지갑 서명 팝업을 닫거나, 지갑 앱이 설치되지 않은 경우.
- **처리**: "지갑 연결에 실패했습니다. NEAR 지갑을 설치하거나 재시도해주세요" Toast 노출 + MyNearWallet 설치 링크 제공. 이전 단계 상태 유지.

### E-2: 파일 형식 미지원 (Step 2)
- **발생 조건**: VCF/PDF/TXT 외의 파일(예: .docx, .jpg)을 드래그앤드롭한 경우.
- **처리**: 업로드 전 클라이언트에서 즉시 차단. "지원 형식: VCF, PDF, TXT만 허용됩니다" 인라인 에러 표시. 서버에 파일이 전송되지 않음.

### E-3: 파일 업로드 실패 (Step 2)
- **발생 조건**: NPC 업로드 중 네트워크 오류 발생.
- **처리**: 3회 자동 재시도(1초, 3초, 7초 간격). 3회 후에도 실패 시 "업로드에 실패했습니다. 파일은 로컬에 보관되어 있으니 나중에 다시 시도해주세요" Toast 노출.

### E-4: TEE 분석 타임아웃 (Step 3)
- **발생 조건**: 분석 요청 후 60초 이내 응답 없음.
- **처리**: Progress 바 정지 + "분석 환경이 혼잡합니다. 잠시 후 재시도해주세요" 안내. TEE 세션 자동 종료, 중간 데이터 강제 소각(Emergency Purge) 수행. 사용자 유전자 파일은 NPC에 그대로 보존.

### E-5: 결제 트랜잭션 실패 (Step 5)
- **발생 조건**: Confidential Intents 트랜잭션 revert.
- **처리**: "결제가 취소되었습니다. 보험사로 어떠한 정보도 전송되지 않았습니다." Dialog 노출. 잔액 즉시 환불 확인 링크(NEAR Explorer) 제공.

## 4. 모바일 UX 시나리오 (Mobile-Specific Flow)

모바일 환경에서는 NEAR 지갑 앱(MyNearWallet, Meteor Wallet)과의 딥링크(Deep Link) 연동이 데스크톱과 다르게 동작합니다.

### 모바일 지갑 연결 (Step 1 모바일)
1. "Connect Wallet" 버튼 클릭 시 `@near-wallet-selector/modal-ui` 모달 노출.
2. 사용자가 설치된 지갑 앱 선택 → 앱 딥링크(`mynearwallet://...`) 또는 WalletConnect QR 코드로 전환.
3. 지갑 앱에서 승인 후 WebSocket 또는 redirect URI를 통해 DApp으로 인증 결과 반환.
4. 앱이 미설치된 경우 해당 지갑의 앱스토어 링크로 유도.

### 모바일 결제 서명 (Step 5 모바일)
1. Dialog 모달 내 "서명하여 결제" 버튼 클릭.
2. 지갑 앱으로 자동 딥링크 전환 → 트랜잭션 서명 화면 표시.
3. 서명 완료 후 DApp으로 redirect → 결제 완료 상태 화면 전환.

### 모바일 반응형 원칙
- Step 3 대시보드의 Grid 레이아웃: 데스크톱 2열 → 모바일 1열 스택.
- Step 4 보험 카드 탭(Tabs): 모바일에서 수평 스크롤 탭으로 전환.
- 파일 업로드(Step 2): 드래그앤드롭 외에 "파일 선택" 버튼 병행 제공 (모바일 파일 피커 연동).

## 5. 성능 기준 (Performance Criteria)

| 단계 | 항목 | 목표 기준 |
| :--- | :--- | :--- |
| Step 1 | 지갑 연결 모달 노출 | < 300ms |
| Step 2 | 파일 업로드 완료 (10MB 기준) | < 10초 |
| Step 3 | TEE 분석 완료 (P50) | < 20초 |
| Step 3 | TEE 분석 완료 (P95) | < 45초 |
| Step 3 | 타임아웃 강제 종료 | 60초 |
| Step 4 | 대시보드 렌더링 (첫 화면) | < 1초 |
| Step 5 | 결제 트랜잭션 확인 (NEAR finality) | < 3초 |

---

## 6. UI/UX 디자인 원칙 (Design Principles)
1. **극적인 투명성 (Radical Transparency)**: 내 데이터가 지금 어디에 머물러 있고, 언제 폐기되는지를 사용자가 상시 모니터링할 수 있도록 알림(Toast/Progress View) 극대화.
2. **법률 용어의 인간화**: 어려운 보험/블록체인 용어 대신 "데이터가 소각되었습니다", "보험사에는 블라인드 처리되었습니다" 등 직관적 워딩 사용.
