## 0. 읽는 법

이 문서는 코딩 중 "이 버튼 라벨 뭐로 하지?", "에러 메시지 어떻게 쓰지?"를 즉흥으로 결정하지 않기 위한 명세서다. 각 화면의 모든 텍스트, 버튼 상태, 에러 메시지가 여기에 정의되어 있다.

**표기 규칙**
- `[KO]` / `[EN]` — 한국어 기본, 영어 병기 (next-intl 키 기준)
- `variant=` — Shadcn 컴포넌트 variant 명
- `disabled` / `loading` / `idle` — 버튼 상태
- `Toast` / `Dialog` / `Inline` — 에러 노출 위치

### 2-3. 프라이버시 서약 (Accordion)

Accordion 헤더: `서비스 이용 전 확인해 주세요 (프라이버시 서약)`

Accordion 본문:
```
1. 귀하의 유전자 파일은 귀하의 개인 금고(NEAR Private Cloud)에만 저장됩니다.
2. 분석은 하드웨어 보안 영역(TEE) 내부에서만 이루어지며, 운영자도 열람할 수 없습니다.
3. 분석 완료 즉시 원본 데이터는 메모리에서 영구 삭제됩니다.
4. 보험사에는 '조건 충족 여부'만 전달되며, 유전자 수치는 절대 공개되지 않습니다.
5. 귀하는 언제든지 데이터를 삭제하거나 서비스를 탈퇴할 수 있습니다.
```

서약 확인 체크박스: `위 내용을 확인했습니다.`

## 4. Step 3 — TEE 분석 (`/analysis`)

### 4-1. 분석 단계별 텍스트 (Progress 바와 동기화)

| 단계 | Progress | 상태 텍스트 | 서브 텍스트 |
|---|---|---|---|
| 1단계 | 0 → 15% | `하드웨어 보안 영역(TEE) 진입 중` | `외부 접근이 차단된 격리 환경을 준비하고 있습니다.` |
| 2단계 | 15 → 40% | `개인 금고에서 암호화 데이터 로딩 중` | `운영자를 포함한 외부 인원은 이 과정에 접근할 수 없습니다.` |
| 3단계 | 40 → 80% | `IronClaw Runtime에서 유전자 데이터 분석 중` | `AI가 위험 프로파일을 생성하고 있습니다. 잠시 기다려 주세요.` |
| 4단계 | 80 → 95% | `보험 적합성 증명(ZKP) 생성 중` | `수치를 노출하지 않는 영지식 증명을 준비하고 있습니다.` |
| 5단계 | 95 → 100% | `분석 완료 — 원본 데이터 소각 중` | `TEE 메모리에서 복호화 데이터를 영구 삭제하고 있습니다.` |

### 4-2. Memory Purge 완료 알림 (Sonner Toast)

```
원본 복호화 데이터가 TEE 메모리에서 영구 소각되었습니다.
분석 소요: {n}초  ·  잔류 데이터: 0 bytes
```
`variant="success"` — 녹색 체크 아이콘, 5초 유지

### 4-3. TEE 상태 배지 (우측 하단 고정)

| 상태 | 텍스트 | 색상 |
|---|---|---|
| 분석 중 | `TEE: ACTIVE` | `text-emerald-400` + 점 깜빡임 |
| 소각 중 | `TEE: PURGING` | `text-yellow-400` |
| 완료 | `TEE: CLOSED` | `text-zinc-400` |

### 4-4. Step 3 에러 메시지

| 에러 상황 | 메시지 | 노출 위치 | 액션 버튼 |
|---|---|---|---|
| TEE 타임아웃 (60초) | `분석 환경이 혼잡합니다. 원본 데이터는 안전하게 보관되어 있습니다. 잠시 후 다시 시도해 주세요.` | Dialog (중앙 모달) | `다시 시도` / `나중에 하기` |
| TEE 내부 오류 | `분석 중 오류가 발생했습니다. 데이터는 즉시 소각되었습니다. 파일은 개인 금고에 그대로 보관됩니다.` | Dialog (중앙 모달) | `다시 시도` |
| ZKP 생성 실패 | `증명 생성에 실패했습니다. 분석 결과는 생성되었으나 보험사 전달 증명을 다시 생성해야 합니다.` | Toast (error) | `재생성` 버튼 |

### 5-2. 위험 프로파일 카드 (좌측 패널)

**카드 헤더**: `유전자 위험 프로파일`
**카드 서브**: `수치는 표시되지 않습니다. 카테고리 수준만 분석됩니다.`

| 카테고리 | 표시명 | 위험 수준별 Badge |
|---|---|---|
| oncology | `종양 / 암` | high: `고위험 Badge(destructive)` / moderate: `주의 Badge(warning)` / normal: `정상 Badge(secondary)` |
| cardiovascular | `심혈관` | 동일 |
| metabolic | `대사` | 동일 |
| neurological | `신경계` | 동일 |

**카테고리별 어드바이저리 메시지**:
`{advisoryMessages.oncology}` 등 TeeAnalysisOutput에서 주입.

**하단 안심 문구**:
```
AI 추천 근거: "해당 유전자 지표 기준 충족"
실제 수치 및 질병명은 이 화면에만 표시되며 외부로 전송되지 않습니다.
```
`text-zinc-400 text-xs`
