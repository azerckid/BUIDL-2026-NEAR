# Antigravity Global Standards & Communication Rules

이 문서는 Antigravity AI 에이전트가 본 프로젝트를 수행함에 있어 준수해야 할 최상위 원칙을 정의합니다.

## 1. 커뮤니케이션 및 승인 원칙
- **No Emojis & No Fluff**: 전문적이고 명확한 텍스트로만 소통한다.
- **선 승인 후 수정**: 코드를 수정하기 전에 반드시 계획을 보고하고 명시적 승인을 얻어야 한다.
- **근거 기반 답변**: 사용자의 질문에 답변할 때는 반드시 도구(Tool)로 현재 상태를 검증한 후 답변한다.
- **자가 점검 강제**: 도구 호출 전 4가지 핵심 질문(설명 여부, 승인 여부, 영향 분석, 의견 교환)에 스스로 답한다.

## 2. 보안 및 데이터 안전
- **데이터 보존 우선**: 파괴적인 명령어 실행 전 반드시 백업을 수행하며, 데이터 유실을 방지한다.
- **환경 격리**: Local과 Production 환경을 엄격히 분리 관리한다.

## 3. 개발 표준 및 품질
- **표준 아키텍처 준수**: 임시 방편을 금지하며, 공식적이고 안정적인 Best Practice를 따른다.
- **영향 범위 분석**: 공통 로직 수정 시 기존 기능에 영향이 없도록 격리하여 구현한다.

## 4. 문서 관리 표준 (365 Principle)
- **5대 디렉토리 구조 엄격 준수**:
    1. `docs/01_Concept_Design/`: 컨셉, 디자인 가이드, 비즈니스 비전.
    2. `docs/02_UI_Screens/`: UI 스크린, 화면 흐름.
    3. `docs/03_Technical_Specs/`: 기술 명세, API, DB 스키마.
    4. `docs/04_Logic_Progress/`: 비즈니스 로직, 상태 관리, 로드맵.
    5. `docs/05_QA_Validation/`: QA 체크리스트, 시스템 검증.

## 5. 단계별 품질 보증 (DoD)
- **통합 사용자 시나리오 테스트**: 각 Phase 종료 전 실환경에서의 정상 작동을 검증한다.
- **완료 보고서**: QA 수행 결과와 해결 내역을 포함한 완료 보고서를 작성한다.

## 6. Git 커밋 컨벤션
- **Conventional Commits**: `type(scope): 설명` 형식을 준수하며 한글로 상세 내용을 작성한다.

## 7. 기술 스택 원칙
- **하향 조정 금지**: 공식 기술 스택(Zod, Luxon, NEAR SDK 등)과 운영 맥락을 100% 유지하며 정확하게 구현한다.
